Код	Название	Когда используется
200	 OK	  Всё успешно, вернули данные (GET, PUT)
201	Created	   Успешно создан ресурс (POST)
204	No Content	  Успешно, но без ответа (например, DELETE)
400	Bad Request	  Неправильный запрос (например, не хватает данных)
401	Unauthorized	  Не авторизован (например, нет токена)
403	Forbidden	  Доступ запрещён (авторизация есть, но прав нет)
404	Not Found	  Ресурс не найден (например, ID не существует)
500	Internal Server Error	  Ошибка на сервере
------------------------------------------------------------------
HTTP — это протокол, по которому клиент и сервер обмениваются данными. Он использует:

GET — получить что-то
POST — создать
PUT — полностью обновить
PATCH — частично обновить
DELETE — удалить
------------------------------------------------------------------
CRUD — это просто:

Create → POST
Read → GET
Update → PUT / PATCH
Delete → DELETE
------------------------------------------------------------------
API — интерфейс для общения
API (Application Programming Interface) — это просто способ, с помощью которого другие программы (или frontend) могут обращаться к твоему бэкенду. Через URL'ы, запросы и ответы.
------------------------------------------------------------------
undefined - значение отсувствует, переменная не инициализирована, нет нужного параметра, функция ничего не возвращает, когда в объекте нет нужного свойства
null - пустое значение, намеренно присвоенное программистом
